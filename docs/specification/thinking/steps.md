## 1. レベル別ステップ構成

### レベル1（難易度1〜5）

```
[Step0] 設問A・B・Cに回答する
        ↓ （採点：80点ルール）
[最終問い] 「つまり、どういうこと？」（難易度2以上のみ）
        ↓ （採点：90点ルール）
[最終フィードバック＋振り返り]
```

設問A・B・Cは1つの画面に並べて表示し、まとめて回答する。

### レベル2（難易度3〜5）

```
[Step0] 各思考タイプの中で「何を確認するか」を定義する
        ↓ （採点：80点ルール / user_core抽出）
[Step1] 定義した切り口に沿って回答する
        ↓ （採点：80点ルール）
[最終問い]「つまり、どういうこと？」
        ↓ （採点：90点ルール）
[最終フィードバック＋振り返り]
```

### レベル3（難易度3〜5）

```
[Step0] どのタイプをどの順序で使うかを選び、理由を述べる
        ↓ （採点：80点ルール / user_core抽出）
[Step1] 選んだ順序で実際に回答する
        ↓ （採点：80点ルール）
[最終問い]「つまり、どういうこと？」
        ↓ （採点：90点ルール）
[最終フィードバック＋振り返り]
```

### レベル4（難易度3〜5）

```
[Step0] 「何を解くべきか」という問いを自分で定義する
        ↓ （採点：80点ルール / user_core抽出）
[Step1] 解くために使うタイプと順序を選ぶ
        ↓ （採点：80点ルール）
[Step2] 選んだ順序で実際に回答する
        ↓ （採点：80点ルール）
[最終問い]「つまり、どういうこと？」
        ↓ （採点：90点ルール）
[最終フィードバック＋振り返り]
```

## 2. 80点ルールの分岐（各ステップ共通）

```
result.score >= 95
  → 問答無用で次のステップへ進む（フィードバックなし）

80 <= result.score < 95
  → 指摘1つを表示
  → 「このまま次へ進む」「修正する」を選択
  → 「修正する」を選んだ場合、回答欄を再表示

result.score < 80 かつ retryCount === 0
  → 問い返しを表示（役割B）
  → retryCount = 1
  → 回答欄を再表示

result.score < 80 かつ retryCount === 1
  → 打ち切りフィードバックを表示
  → 次のステップには進まない
  → done = true
```

打ち切りで終了した場合、最終問いは表示しない。

**設計意図：**
- 95点以上でフィードバックなしにする理由：完璧な回答に余計な指摘を加えると学習体験を損なうため
- 80〜95点で1つのみ指摘する理由：「直すか次へ進むか」の選択権をユーザーに与えることで、自分の判断で前に進む体験を提供する
- 2回目の80点未満で打ち切りにする理由：同じステップで3回以上ループすると学習より疲弊が勝るため

## 3. 90点ルールの分岐（最終問い）

```
result.score >= 90
  → そのまま最終フィードバックへ

result.score < 90 かつ finalRetried === false
  → 1回だけ問い返しを表示
  → finalRetried = true
  → 再回答欄を表示

result.score < 90 かつ finalRetried === true
  → 最終フィードバックへ（打ち切りではなく通常フィードバック）
```

**80点ルールと異なり90点ルールは打ち切りにしない理由：**  
最終問いは問題全体の統合であり、ここで打ち切りにするとセッション全体が無効になる印象を与える。
再回答後は点数によらず最終フィードバックへ進む。

## 4. 振り返りエリアのUI

`ENABLE_REFLECTION === true` のときのみ表示する。

```
[D1の問いかけ]（最終フィードバックの末尾に既に表示済み）
  ↓
[振り返り回答欄1] テキスト入力
  ↓ ユーザーが回答 or スキップ
  - 回答 → 「振り返りに回答する」ボタン → 役割D2のAPI呼び出し
  - スキップ → セッション終了（振り返りエリアを display:none）

[D2のフィードバック＋深掘り問いかけ]
  ↓
[振り返り回答欄2] テキスト入力
  ↓ ユーザーが回答 or スキップ
  - 回答 → 「さらに回答する」ボタン → 役割D3のAPI呼び出し
  - スキップ → セッション終了

[D3のフィードバック]（問い返しなし・示唆で締める）
```

## 5. 各ステップのmode値

| mode | 使うレベル | 内容 |
|---|---|---|
| `'answer'` | レベル1（Step0）・レベル2〜4（最終ステップ） | 思考タイプへの回答 |
| `'define'` | レベル4（Step0） | 「何を解くべきか」の問い定義 |
| `'typeselect'` | レベル3（Step0）・レベル4（Step1） | タイプ選択と順序 |
| `'define2'` | レベル2（Step0） | 各タイプで何を確認するかの定義 |

user_coreの抽出は `mode !== 'answer'` のステップでのみ行う。
