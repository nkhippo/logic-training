# ADR-0004: AI モデルを Claude Sonnet 4.6 系に統一

**日付**: 2026-05-25
**ステータス**: Accepted
**決定者**: Naoya

---

## コンテキスト

問題生成と採点で求める品質が異なるため、運用時にモデルと温度設定を統一方針として明示する必要があった。

## 決定内容

主力モデルを Claude Sonnet 4.6 系に統一し、生成系と採点系で温度設定を分ける運用とする。

## 選択肢と却下理由

| 選択肢 | 却下理由 |
|---|---|
| タスクごとにモデルを固定しない | 再現性・品質管理が難しく運用判断が分散する |
| Claude Sonnet 4.6 系に統一（採用） | 品質と運用一貫性を両立しやすい |

## 結果・影響

モデル選定の判断軸が統一され、仕様書でのパラメータ設計意図が追跡しやすくなった。
