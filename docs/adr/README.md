# ADR (Architecture Decision Record)

このディレクトリは、設計判断を「1判断1ファイル」で管理するための正本です。
背景・選択肢・却下理由・影響を時系列で追える状態を維持します。

---

## 追加ルール

- ファイル名は `NNNN-topic.md` 形式（例: `0006-example-decision.md`）
- 1ファイルにつき1つの意思決定のみを記録する
- 既存判断の更新は「追記」ではなく新しい ADR を追加して関係を明記する
- ステータスは `Accepted / Superseded / Deprecated` を使う

---

## フォーマット

```markdown
# ADR-XXXX: （タイトル）

**日付**: YYYY-MM-DD
**ステータス**: Accepted / Superseded / Deprecated
**決定者**: Naoya

---

（なぜこの判断が必要だったか・背景）

## 決定内容

（何を選択したか）

## 選択肢と却下理由

| 選択肢 | 却下理由 |
|---|---|
| （案A） | （理由） |

## 結果・影響

（この決定によって何が変わったか・トレードオフ）

## 関連ドキュメント

- （関連ファイルへのリンク）
```
