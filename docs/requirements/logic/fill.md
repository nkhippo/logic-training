# 穴埋めタブ — 要件定義

**タブID**: fill  
**バージョン**: 1.0  
**作成日**: 2026-05-25

> 共通要件は `requirements/common.md`、論理トレーニング全体は `requirements/logic/overview.md` を参照。

---

## 1. このタブの目的

接続詞を文脈から選ぶことで、文と文の論理的接続を精確に読み取る力を鍛える。

**主に鍛える力**: 読む力（A）

接続詞は文章の論理構造を最も直接的に示す要素である。
「しかし」「だから」「つまり」などの接続詞を正しく選べることは、
前後の文の論理関係（逆接・因果・要約など）を正確に把握していることを意味する。

---

## 2. ユーザー視点での動作

1. テーマ（文書タイプ）・業界・難易度を選択する
2. 「問題を生成する」を押すと、接続詞を空欄にしたビジネス文書が表示される
3. 各空欄に当てはまる接続詞を選択または入力する
4. 「答え合わせ」を押すと採点フィードバックが表示される
5. 問題と採点結果を過去問として保存できる

---

## 3. 難易度別の学習体験

| 難易度 | 想定する学習体験 | ヒントの有無 |
|---|---|---|
| 1（入門） | ごく少数の接続詞を文脈から選ぶ。接続詞の種類（逆接・因果など）のヒントが与えられる | あり |
| 2（基礎） | 接続詞の数が増え、軽いヒントの中で選択する | 軽いヒント |
| 3（標準） | ヒントなしで接続詞を文脈から推論する | なし |
| 4（上級） | 接続詞の数がさらに増え、複数の論理関係を同時に判断する | なし |
| 5（超難問） | 最多の接続詞数で、文章全体の論理構造を完全に把握する必要がある | なし |

**設計意図：**
- 全難易度で文章量を固定（接続詞の選択に集中させるため）
- 難易度の差は「穴抜き数」と「文脈の複雑さ」で表現する
- ヒントを入門・基礎のみに提供する理由：初学者が「なぜこの接続詞か」の考え方を学ぶ足がかりにするため

> 具体的な実装値（文字数・穴抜き数）は `specification/logic/fill.md §2-1` を参照。

---

## 4. 採点フィードバックの要件

採点後に以下を表示すること：

- **完全補完文**：全接続詞を正解で補完した版
- **各設問への日常例文**：その接続詞が日常でどう使われるかを示す
- **代替接続詞を使った場合の論旨変化**：別の接続詞を選んだ場合に文章の意味がどう変わるかを示す

**設計意図：**  
正誤だけでなく「なぜその接続詞か」を理解させることが目的。
代替接続詞の論旨変化を示すことで、接続詞の選択が文章全体の論理に与える影響を体感できる。

---

## 5. 回答方法

- テキスト入力のみ（写真提出は対象外）
- **写真対応しない理由**：穴埋め問題は短い選択・入力であり、写真添削に足りる情報量がない
