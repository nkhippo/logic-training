## 根本原因カテゴリ定義

| カテゴリ | 意味 |
|---|---|
| `仕様書の粒度不足` | 数値・条件式・JSON例がなく、Cursorが解釈を誤った |
| `移植漏れ` | legacy → React 移植時に機能・ロジックが落ちた |
| `指示書の欠落` | Cursor指示書がなかった、または不十分だった |
| `設計ドキュメント未更新` | コードと仕様書・要件定義書が乖離していた |
| `テスト不足` | 完了定義の確認が甘く、動作未検証のまま通過した |
| `その他` | 上記に当てはまらない |


## BUG-2026-05-27-001: 思考トレーニング機能の大幅欠落（Phase 2-2）

| 項目 | 内容 |
|------|------|
| Issue | Phase 2-2 Opusレビューで発覚（Issue未起票） |
| PR | Phase 2-2 React移行PR |
| 発生箇所 | `src/logic/thinkingLogic.js` / `src/components/thinking/` |
| 事象 | Vanilla JS版（1289行）に存在した思考トレーニングの主要機能がReact版（202行）で軒並み消滅。80点ルール・役割A/B/C・振り返りD1/D2/D3・90点ルール・打ち切りロジックすべてが未実装 |
| 直接原因 | Cursorが移植元ファイル（`legacy/js/thinking/app.js`）を全体通読せず、一部のみを参照して実装した |
| 根本原因 | ①Cursor指示書が存在しなかった ②「機能等価で移植」という前提が指示に明記されていなかった ③完了定義が「動作する」止まりで機能単位のチェックリストがなかった |
| 根本原因カテゴリ | `指示書の欠落` / `移植漏れ` |
| 再発防止策（実施済み） | CLAUDE.md「Cursor指示書の品質基準」に移植対照表必須・移植元全体通読指示・機能単位完了定義を追加。dev-flow.mdcに設計ドキュメント整備フェーズ（Step 3）を追加 |
