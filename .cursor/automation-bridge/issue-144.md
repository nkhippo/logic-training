## Body

## 背景・目的

現在のIssueテンプレートでは「完了定義」をチェックボックス形式で記載しているが、
以下の問題がある：

1. **自動化との相性が悪い** - GitHub APIはIssue本文をまるごと書き換えるしかなく、チェックボックスの自動チェックは本文破損リスクがある
2. **Cursorが完了確認できない** - チェックボックスはレビュワーが手動でチェックするものであり、Cursorが「完了したか」を判断する基準として機能していない
3. **意図が伝わりにくい** - チェックボックスの羅列より、「この状態になったら完了」という文章の方がCursorへの指示として明確

## 変更方針

### Issue本文・CLAUDE.mdの完了定義フォーマットを変更

**変更前（チェックボックス形式）：**
```markdown
## 完了定義
- [ ] .github/labeler.yml が新規作成されている
- [ ] テスト用に任意のブランチからPRを作成し、ラベルが自動付与されることを確認
```

**変更後（文章形式）：**
```markdown
## 完了定義
以下の状態になっていること：
- `.github/labeler.yml` が新規作成されている
- テスト用PRを作成したとき、ファイルパスに応じたラベルが自動付与される
- 既存の `waiting-for-review` 付与フローが引き続き動作している
```

### 変更対象ファイル

1. `CLAUDE.md` - Issue起票ルールの「完了定義」記載例をチェックボックスなしに変更
2. `.github/ISSUE_TEMPLATE/cursor_task.md`（#133で作成予定）- 完了定義セクションをチェックボックスなしに変更
